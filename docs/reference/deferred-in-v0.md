# Deferred in v0

## Who should read this page

Program leadership, facilitators, and SMEs tracking what is intentionally out of scope for the first release.

## Purpose

Record what is deliberately deferred in the HERO v0 / pilot build, so users know where documentation is incomplete by design.

## Deliberately deferred

- Final tab-by-tab Power BI documentation for the **AIM Shipment Revenue** and **POS Glidepath** views.
- Final **cluster-specific permission matrix** by Business Unit, Forecast Partner, and Brand.
- Any final changes to **enrichment-type taxonomy** before pencils-down.
- Any **pilot-to-production differences** in export-review cadence or freeze rules.

!!! note "AIM and POS"
    AIM = the Shipment Revenue view; POS = Point of Sale (sell-through) used in the Glidepath view. Detailed documentation is owned outside this manual for v0.
